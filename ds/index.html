字級使用紀律（MUST）——實測常見「用錯層級」，一律避免：
· 內文一律 body；small 只在表格附註 / 標籤等特殊情境用，不可濫用當內文。
· 禁止拿 body 當標題（標題用 h1~h4），禁止拿 note / small 當內文——這兩種最常見。
· note（最小級）僅用於圖片說明、footer、版權，不是內文層級；活動頁預設不使用 note。

二、不誤導點擊

  • Hero 內不放 CTA 按鈕——避免使用者還沒看內容就被誘導跳轉，說服不足就跳走反而轉換差；CTA 放內容區塊之後。
  • 任何標題、裝飾文字不做按鈕樣式（不加背景色塊、邊框、Pill 膠囊形狀），怕被誤認可點；要強調用字級 / 字重 / 顏色。
  • 沒有連結或互動的卡片 / 區塊，一律不加 hover 效果（不變色、不浮起、不變邊框、不放大）。

同區塊對齊規則

不同 section 容器寬度可不同，但同一 section 內所有元素容器寬度必須對齊一致。寫一個 section 時自問：

  • 這 section 用了幾種容器寬度？（理想 1 種，最多 2 種，第 2 種須為「視覺層」如大圖、banner）
  • 同類元素（標題、內文、卡片、按鈕）左右邊界是否完全切齊？
  • 不齊時差距 ≥ 64px（可接受、讀者理解為刻意）還是 < 32px（「邊界靠近但沒對齊」，比明顯不同還醜，必須改）？

表格 mobile 卡片化

桌機保留完整 <table> 結構；mobile 一律卡片化（每 row 變獨立卡片、label/value 對照），不用橫向滾動。

四、文案品質

禁用類型說明 / 改寫方向
空洞行業術語「賦能」「閉環」「抓手」「下沉」
過度稱讚式「完美的」「極致的」「革命性的」
二分法行銷金句「選擇 X 或 Y」「看見/看不見」「不只是 X 更是 Y」→ 改寫成具體事實（如「每批檢驗 100+ 種農藥殘留」）
過長標題 > 20 字、段落 > 4 行
強調大字（font-size ≥ 22px 的獨立段落）：內文 ≤ 40 字、加寬 letter-spacing 0.04~0.08em 與 line-height 1.4~1.6、每 section 最多一個、用 <br> 強制斷在逗點/句點後，不可斷在詞中（「不容妥協」須同一行）。
Statement Section 判斷（命中 3 條以上即禁用）：① 整段只有一句話 ② 字級 ≥ 22px ③ 容器 ≥ 880px ④ 上下 padding ≥ 96px ⑤ 內容是空洞金句。例外：活動頁主視覺標語經明確標註強化主旨可用，並註記破例原因。

六、AI 友善（SEO / AEO）

讓 AI 搜尋引擎（ChatGPT、Perplexity、Gemini）能正確理解引用。核心三條：結構清楚 + 內容明確 + 機器可讀。

語意化 HTML

  • 每頁一個 <h1> 含核心關鍵字
  • H2~H6 嚴格遞進、不跳級、不為視覺用 heading
  • 用 semantic tag / ul / table / blockquote，不用 div 模擬

內容結構（AEO）

  • 問答式：問題當 H2/H3、下接答案
  • 倒金字塔：結論先行
  • list / table 結構化 + 具體數據

Metadata / Crawler

  • title 60 字 / description 155 字 / OG 四項
  • Schema.org JSON-LD（Product / Article / FAQPage / BreadcrumbList）
  • robots 開放 AI bot、sitemap 月更、SSR 直出、3 秒首屏

七、AI Slop 禁用清單（10 條）

無論什麼主題都不准出現，活動頁再有趣也不例外。

九、三層協作與衝突規則

任何頁面 = 核心（全 MUST）
形象頁   = 核心 + 形象頁延伸（穩定鎖定值）
活動頁   = 核心 + 活動頁延伸（少數建議）+ 該檔風格參考素材（每檔外掛）

衝突優先序：  核心護欄 (MUST)  >  該檔風格參考素材  >  延伸的 SHOULD / MAY 建議
白話：護欄永遠贏；護欄之內，新素材贏過預設建議。
Layer 2 — 頁面規範

形象頁規範 / 活動頁規範 （擇一套用）

這裡才是「頁面實際長怎樣」的規範，跟核心一樣重要。兩者互斥，切換下方分頁看各自規範——做一個頁面時只會用到其中一頁。

一次只會套用一種延伸 · 目前顯示：活動頁

無毒農品牌門面的固定長相，風格穩定、低頻更新。這是一組鎖定值，不需外掛素材，直接照做。 完整文件：spec-image-page.md ↗

新建頁面前置：先給參考，否則走 Wireframe
形象頁雖有固定鎖定值，但版面編排仍需依據。做新建網頁時——請提供參考網站或參考圖檔（既有類似頁 / 設計稿 / 競品截圖皆可）。 若完全沒有參考，AI 會以 Wireframe（線框稿）製作：只排版面結構與區塊層次、不做視覺完稿，待你補參考再完稿。避免在沒依據下自行發明不符品牌的版面。
字級層級：形象頁會用到 note（14/300） 的地方＝圖片說明文字、footer、版權；其餘一律 body。內文不可用 note / small、標題不可用 body，small 僅特殊情境（表格附註 / 標籤）。

活動頁的第一件事 · 不可略過
先拿到「風格參考素材」，才開得了工

這份 Design System 的任務是確保風格不偏離無毒農形象太多，它無法保證頁面變有趣。 所以要從頭設計一份新的活動頁、想讓畫面有趣時，請另外提供參考網站或圖片素材，AI 會依參考抓取視覺風格與設計元素。沒有參考，AI 只能產出偏形象頁的保守版本，做不出活動該有的氣氛。

開工前必備（AI 會主動跟你要，缺了不動工）
  • moodboard 或參考風格網頁 URL
  • 該檔主題色（主色 + 輔色）
  • 關鍵視覺元素（主視覺圖 / 插畫風 / 質感傾向）
  • 字型傾向（若有；主字 Noto Sans TC 不可換）

風格參考協議（AI agent 必做）

  1. 載入核心護欄（不可違背）。
  2. 檢查該活動是否已有風格參考素材（見上方必備清單）。
  3. 沒有素材 → 主動提醒使用者提供，並說明長相由素材決定、不給只能做保守版。
  4. 依「護欄 + 素材」推導出該檔專屬規範（專屬色、圓角、間距、動態量等），存進活動資料夾 / LP brief 供複用。
  5. 衝突：素材 vs 建議 → 以素材為準；素材 vs 核心護欄 → 以護欄為準、退回素材並向使用者說明。
