## 規範哲學

C 方案 = **文青風格 × 主流 AI 預設框架**。

- **主色（含子品牌）完全延續無毒農既有品牌色**，不另創新色
- 加 2 個**延伸色**（深森林綠、極淺綠）給 C 風格的標題強調與區隔背景用
- **字形 / 版型 / 間距 / 比例 / 攝影風格**做出差異化，營造文青質感
- 對齊**主流 AI 在「無提示」時自然會生成的設計風格**：襯線標題 + 大留白 + 0 圓角 + 大圖

→ C 是「**換包裝**」（主色不變、版型字型重做）。

## 六、圓角與間距

### 圓角

採用 **0px**（極簡文青風）。例外：頭像 / 圓形圖示用 50%（圓形）。

### 間距

基本單位：**4px**（強制 4px 倍數，常用值偏大）

| Token | 值 | 用途 |
|---|---|---|
| xs | 4px | 圖示與文字之間 |
| sm | 8px | 緊密區塊 |
| md | 16px | 段落內元素 |
| lg | 24px | 段落間 |
| xl | 32px | 區塊內子段落 |
| 2xl | 48px | Section 內主分隔 |
| 3xl | 64px | 中等 Section 之間 |
| 4xl | **96px** | 大 Section 之間（**常用**） |
| 5xl | **112~128px** | 主 Section padding（**桌機常用**） |
| 6xl | **160~192px** | 極寬鬆呼吸（用於 Hero 與下一段之間）|

Section padding：桌機 **112~128px** / 手機 **64~80px**。

## 七、RWD 斷點與調整策略

### 三段斷點

| 斷點 | 範圍 | 對應裝置 |
|---|---|---|
| mobile | `< 768px` | 手機 |
| tablet | `768px ~ 1199px` | 平板、小桌機 |
| desktop | `>= 1200px` | 桌機、寬螢幕 |

```scss
.element { /* mobile */ }
@media (min-width: 768px) { .element { /* tablet */ } }
@media (min-width: 1200px) { .element { /* desktop */ } }
```

→ Mobile-first 寫法。

### 跨版型 RWD 共通策略

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 容器 max-width | 全寬 - 32px | 全寬 - 64px | **880px** |
| 內容左右 padding | 24px | 48px | 用 max-width |
| Section 上下 padding | 64~80px | 96px | **112~128px** |
| h1 字級 | 32px | 40px | 48px |
| h2 字級 | 26px | 32px | 36px |
| body 字級 | 16px | 17px | 18px |
| 按鈕 padding | 14px 28px | 16px 32px | 16px 32px |
| 主 nav | 漢堡選單 | 漢堡或展開 | 橫向展開 |
| 圖片裁切 | 4:5 / 4:3 | 4:5 / 4:3 | 4:5 / 16:9 / 21:9 |

## 八、整體排版策略

### 1. 頁面容器策略

| 頁面類型 | 容器策略 |
|---|---|
| 商品列表 / 詳情 | max-width **880px** 置中 |
| Blog 列表 / 文章 | max-width **720px** 置中（更窄、更精緻） |
| 訂單 / 結帳 / 表單 | max-width 524px 置中 |
| **會員中心** | **不受寬度限制（全寬）**——需要橫向空間放 sidebar + 主內容 |
| Landing / 子品牌 | 全寬，內部各區塊 max-width 880px |

### 2. 懸浮元素（**極簡：盡量不用**）

| 元素 | 規範 |
|---|---|
| 主 FAB | **不用**（C 文青風不適合）|
| 錨點導航 | 用標題滾動代替 |
| Sticky 底部 CTA | 僅商品詳情頁可用，主綠底白字（不用純黑）|
| 主 nav | 頂部 sticky，下捲淡出 |

### 3. 表格處理

桌機保留 table 結構；mobile 一律**卡片化**（不用橫向滾動，文青風適合卡片）。

### 4. Modal

| 屬性 | 規格 |
|---|---|
| 遮罩 | `rgba(0,0,0,0.5)` |
| 容器 | max-width 480px / mobile 全寬 - 32px |
| 圓角 | **0px** |
| Padding | 32px |
| 關閉 | X 圖示在右上 + 點遮罩可關 |

### 5. 通知 Toast

| 屬性 | 規格 |
|---|---|
| 位置 | 頁面**底部置中**（C 風格不放右上） |
| 寬度 | 桌機 480px / mobile 全寬 - 32px |
| 圓角 | **0px** |
| 動畫 | fade-up 滑入 |
| 自動消失 | 成功 3 秒 / 錯誤手動關閉 |

## 九、常用版型

保留 **8 個本質性版型**。前 5 個是頁面層級；後 3 個（圖文搭配 / 圖片段落 / 圖片按鈕組）是區塊層級，可組合進前 5 個。

### 1. 列表頁

用於：商品列表、Blog 列表、訂單列表、會員紀錄等

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 欄數 | 1 | 2 | **2**（C 用 2 欄不用 3 欄，每張卡更大） |
| 卡片比例 | 4:5 直立 | 4:5 直立 | 4:5 直立 |
| 卡片 gap | 24px | 32px | **48px** |
| 商品名字級 | 18px | 18px | 20px |

### 2. 詳情頁

用於：商品詳情、Blog 文章、活動詳情

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 主 layout | 上下堆疊 | 上下堆疊 | **上下堆疊**（C 不用左右分欄）|
| 圖片 | 100% | 100% | 100% 全寬 |
| 內容 | 下方單欄 max 720px | 同 | 同 |
| 主 CTA 位置 | sticky bottom | inline | inline |

### 3. 表單頁

用於：訂單確認、結帳、會員資料、聯絡我們

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 容器寬度 | 全寬 - 32px | 524px 置中 | 524px 置中 |
| 欄位 layout | 單欄 | 單欄 | 單欄（部分雙欄如姓名/電話） |
| 欄位 gap | 12px | 16px | 16px |
| 送出按鈕 | 全寬 | 自動寬 | 自動寬 |

### 4. Landing Page

用於：活動專頁、子品牌（粥寶寶 / 綠拿鐵 / 等家寶寶 / 益菓保）、首頁、未來新版介紹頁

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| Hero 高度 | 76vh | 84vh | **84~90vh** |
| Hero 文字位置 | 置中 | 置中 | 置中或左下 |
| 區塊堆疊 | 單欄 | 單欄 | **大量單欄堆疊** |
| 區塊間距 | 64px | 96px | **128~160px** |
| CTA 按鈕 | 全寬 | 自動寬 | 自動寬，可並排多顆 |

### 5. Banner（Slider）

用於：首頁 Hero、商品列表 Hero、活動主視覺、子品牌 Hero

| 屬性 | mobile | desktop |
|---|---|---|
| **比例** | **1:1 方形**（維持現況） | **21:9 寬螢幕**（C 拉寬） |
| 容器寬度 | 全寬 | **全寬不收**（不限 1140px）|
| 左右箭頭 | 顯示 | 顯示 |
| **圓點位置** | **圖片外下方獨立區域**（不疊在圖片上） | 同 |
| 圓點數量 | ≤ 8 個 | ≤ 8 個 |
| 自動播放間隔 | 7 秒 | 7 秒 |
| 切換動畫時間 | 300~500ms | 300~500ms |

### 6. 圖文搭配（Media + Text）

用於：產品介紹、品牌故事、產地報導、Landing Page 中段。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 主 layout | 上下堆疊（圖→文）| 上下堆疊 | **左圖右文 60/40 或 40/60**（C 不對稱、留白多）|
| 圖片比例 | 4:3 或 4:5 | 4:5 | **4:5 直立**（呼應商品卡）|
| 圖文間距 | 24px | 32px | **64~96px**（中間 gap，比一般大）|
| 文字 padding | 24px | 32px | 48px |
| 多段交替 | 全部圖在上 | 全部圖在上 | 左圖右文 / 右圖左文交替 |

→ **圓角 0px**；標題 h3 28px Serif；內文 body 17~18px；圖片用 `<figure>` + `<figcaption>` 寫圖說（Serif italic、灰字）。

#### 6.1 適用門檻（避免圖文比例失衡）

4:5 直立圖在桌機高度 = 容器寬度 × 1.25，比想像中高。若右欄只放「標題 + 兩行 intro」，圖把版面撐開後右側留白會異常醜。

| 右欄內容量 | 建議版型 |
|---|---|
| **多**（≥ 1 個 sub-standard 或 ≥ 5 段內容） | **4:5 split-block**（標準圖文搭配，圖文等高自然平衡） |
| **少**（標題 + intro + 短列表，≤ 3 段內容） | **改用全幅 21:9 橫幅 + 集中窄欄**（不硬塞 split） |

→ 內容單薄時硬用 4:5 split 會被讀作「為了用版型而用」，反而破壞文青質感。

#### 6.2 Sticky 圖片黏滯規則

當右欄內容很長（捲動超過一個圖高度），桌機可讓圖 sticky 黏在欄位內、跟著文字滾動，避免「圖看完就消失」。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| `.split-image` position | **`static`**（必須關掉 sticky）| static | **`sticky` / top: 80px** |

→ **手機版必須關 sticky**：split-block 在 `< 900px` 變單欄上下堆疊，sticky 會讓圖在滾動時黏住、疊到下方文字上，造成字疊圖。
→ CSS 範例：

```scss
.split-image {
  position: sticky;
  top: 80px;
}
@media (max-width: 900px) {
  .split-image {
    position: static;
    top: auto;
  }
}
```

### 7. 圖片段落（Image Block）

用於：產地實景、檢驗實驗室、團隊照、產品大圖、品牌氛圍圖。

| 形式 | mobile | tablet | desktop |
|---|---|---|---|
| 單張大圖（feature）| **全寬 21:9**（C 加寬） | 全寬 21:9 | **全寬 21:9**（不限 1140px）|
| 雙圖並排 | 上下堆疊 | 雙欄 4:5 | 雙欄 4:5 |
| 三圖網格 | 上下堆疊 | 雙欄 + 一 | 三欄 4:5 |
| 四圖網格 | 雙欄 1:1 | 雙欄 1:1 | 四欄 1:1 |
| 圖片 gap | 12~16px | 24px | **48px**（C 大留白）|
| 圖說 caption | 圖下 16px、`note 13px`、Serif、灰字 | 同 | 同 |

→ **圓角 0px**；用 `<figure>` + `<figcaption>` 包裝；圖片 alt 必填。

### 8. 圖片按鈕組（Image Card Buttons）

用於：分類入口、子品牌入口、活動入口、首頁 CTA 區。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 卡數 | 1（縱列）| 2 | 3~4 |
| 圖片比例 | 4:5 | 4:5 | **4:5 直立**（C 直立式）|
| 文字位置 | 圖下 | 圖下 | **圖下卡片式為主**（C 不推 overlay）|
| Hover | 圖片淡淡淡入暗（不 zoom）| 同 | 同 |
| 字型 | Sans 標題 + Serif italic 副標 | 同 | 同 |
| 圓角 | **0px** | 0px | 0px |
| 卡片 gap | 24px | 32px | **48px** |

C 推薦變體：

**圖下卡片式**（C 唯一推薦，AEO 友善）
- 圖（4:5 直立）→ 標題（h4 22px Sans Medium）→ 副標（small 15px Serif italic 灰字）→ CTA「了解更多 →」（文字底線型）
- **不用 overlay 變體**：C 風格忌諱文字壓在圖上（影響閱讀、降低文青質感）

→ 整張卡 `<a>` 包裹，整張可點擊；hover 圖片變暗 5%（不 zoom）。

## 十四、AI 友善網站（SEO / AEO）

讓 AI 搜尋引擎（ChatGPT、Perplexity、Gemini 等）能正確理解、引用網站內容。原則：**結構清楚 + 內容明確 + 機器可讀**。

C 方案**天然符合** AI 預設框架（大留白、Serif 標題、單欄堆疊）。

### 1. 語意化 HTML 結構

| 規則 | 說明 |
|---|---|
| 每頁只有一個 `<h1>` | 且包含核心關鍵字（商品名 / 類別名 / 品牌名）|
| H2~H6 嚴格層級遞進 | 不跳級（不能 H1 → H3）；不為視覺用 heading |
| 區塊用 semantic tag | `<article>` / `<section>` / `<nav>` / `<main>` / `<header>` / `<footer>` / `<aside>` |
| 列表用 `<ul>` / `<ol>` | 不用 `<div>` 模擬列表 |
| 表格用 `<table>` | `<thead>` `<tbody>` 完整；不用 div grid 假表格 |
| 引述用 `<blockquote>` | 不用單純 italic / 引號 |

### 2. Heading 內容原則

- **H1**：頁面主標題，包含核心關鍵字
- **H2**：主要區塊標題
- **H3**：子區塊
- 視覺需要大字但語意上不是標題 → 用 `<p class="text-large">`，**不要用 heading 標籤**

### 3. 內容結構（AEO 友善）

| 原則 | 做法 |
|---|---|
| 問答式 | 用問題當 H2/H3，下方接答案 |
| 倒金字塔（結論先行） | 段落第一句寫結論，後面補充細節 |
| 用 list / table 結構化 | 重點資訊用 `<ul>` 或 `<table>`，AI 容易解析 |
| 具體數據 | 用具體數字、單位、日期 |
| 避免空洞行銷 | 不寫「最好的」「業界第一」「革命性」 |

### 4. Metadata 與結構化資料

| 項目 | 規則 |
|---|---|
| `<title>` | 60 字內，包含主關鍵字 + 品牌名 |
| `<meta name="description">` | 155 字內，自然描述頁面，不堆關鍵字 |
| Open Graph | og:title / og:description / og:image / og:url |
| **Schema.org JSON-LD** | 商品 → `Product`；文章 → `Article`；組織 → `Organization`；FAQ → `FAQPage`；麵包屑 → `BreadcrumbList` |

### 5. 圖片 alt 文字

- 內容圖：alt 描述圖片內容
- 商品圖：alt 含商品名 + 主特徵
- 裝飾圖：`alt=""`（空字串）
- 避免：「圖片」「banner」「圖 1」這種無意義 alt
- 避免堆關鍵字

### 6. URL 與內部連結

- URL 簡短、語意清楚
- 內部連結 anchor text 描述性（避免「點這裡」）
- 麵包屑導覽 + Schema BreadcrumbList

### 7. AI Crawler 設定

| 項目 | 規則 |
|---|---|
| robots.txt | 開放主流 AI bot：`GPTBot` / `ClaudeBot` / `PerplexityBot` / `Google-Extended` / `CCBot` |
| sitemap.xml | 完整列出所有公開頁，每月更新 |
| SSR / 靜態化 | **重要內容直接出現在 HTML**，不等 JS 渲染 |
| 頁面載入速度 | 3 秒內首屏可見 |

### 核心原則三條

1. **結構清楚**：H1/H2/H3 階層 + semantic HTML
2. **內容明確**：問答式 + 倒金字塔 + 具體數據
3. **機器可讀**：Schema.org + alt + 不藏在 JS

## 十六、動畫 / 過場原則

| 場景 | 規範 |
|---|---|
| Section fade-in | scroll-triggered，淡入（300ms ease-out）|
| 圖片 lazy load | fade-in（200ms） |
| Banner slider | fade 切換（不用 slide） |
| Hover | 顏色 / 透明度變化（200ms） |
| **不使用** | 彈跳、視差捲動、過度 fade |

→ C 比較強調動畫，但**節制**。原則：不打擾閱讀。
→ 必須支援 `prefers-reduced-motion` 媒體查詢。

## 二十、不可用版型（明確禁止）

C 風格的精神是「**減法**」，不確定要不要留的元素，先砍。

### 視覺類禁用

1. **滿版彩色漸層 banner**（紫到藍、彩虹漸層等套版方向）
2. **圓角卡片 + 彩色左邊框 + icon + 標題 + 描述四件組**（一眼 AI Slop 模板）
3. **任何 emoji 裝飾**（🚀✨💡 當項目符號或標題開頭）
4. **線條風格 SVG 插圖**（機器人、燈泡、齒輪等抽象示意）
5. **fill icon**（C 用 thin line）
6. **假數據過度整齊**（`$9,999`、`99.9% uptime`、`1,234 users` 這種 demo 味）

### 結構類禁用

7. **多欄密集網格**（C 用單欄堆疊；列表頁最多 2 欄）
8. **彩色情境色濫用**（C 以主綠 + 中性色為主，情境色節制）
9. **不規則比例的圖片裁切**（4:3 / 16:9 / 4:5 / 21:9 / 1:1 以外的奇怪比例）
10. **過度裝飾邊框**（雙線、虛線、漸層邊框）
11. **浮動 Tooltip**（除既有 hover 簡述外的彈出框）
12. **5px、7px、10px 等非 4px 倍數的間距**
13. **4px、6px、8px 圓角**（C 統一 0px）
14. **FAB / 過多懸浮按鈕**（極簡）
15. **圖片按鈕 overlay 變體**（文字壓圖上影響閱讀，C 只用「圖下卡片式」）

### 文案類禁用

16. **空洞行業術語**（「賦能」「閉環」「抓手」「下沉」）
17. **過度稱讚式文案**（「完美的」「極致的」「革命性的」）
