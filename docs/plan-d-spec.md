## 規範哲學

D 方案 = **C 的克制美學 ×「不誤導點擊」的視覺收斂**。

- **主色（含子品牌）完全延續無毒農既有品牌色**，不另創新色
- 字型回到**單一 Sans 系**（不再用 Serif 標題），字重整體變細以保留呼吸感
- 視覺骨架沿用 C（0 圓角、大留白、大圖、單欄堆疊）
- 收斂掉容易**誤導點擊**或**轉換閱讀節奏**的元素：Hero CTA、按鈕樣標題、無連結 hover、單字裝飾、Statement Section
- 移除「整片深綠 panel」等過度戲劇化的對比區塊

→ D 是「**C 的踏實版本**」：留下 C 最對的部分，砍掉設計實做後發現會干擾使用者的地方。

| 攝影風格 | 極簡背景、自然光、低飽和 |
| 懸浮 FAB | 不用 |
| 動畫 | fade-in、scroll triggered（節制）|

## D vs C 主要差異速覽

| 項目 | C 文青 | D 精煉 |
|---|---|---|
| 標題字型 | Noto Serif TC | **Noto Sans TC**（統一 sans）|
| 字重基準 | 標題 500、內文 400 | **標題 500/400 混用、內文 300**（更細）|
| 字級 h1 | 48px | **39px**（收斂一階）|
| 延伸色 | 深森林綠 + 極淺綠 | **只留極淺綠**（深森林綠移除，禁用深綠 panel） |
| Hero CTA | 有按鈕 | **無按鈕**（避免未看完就跳走） |
| 標題裝飾 | 大編號、單字裝飾、Serif italic figcaption | **不做按鈕樣標題、不用單字裝飾、預設無 figcaption** |
| Banner 切換 | 左右箭頭 + 圓點 | **僅圓點 + 自動換頁** |
| Statement Section | 容許（大字短文金句區） | **預設禁用**（除非主旨真的需要強化） |
| 卡片 hover | 圖暗、邊框變綠等 | **僅可點卡才有 hover** |
| 同區塊對齊 | 較鬆 | **嚴格要求對齊規則一致** |

## 四、字級階梯

### 字體（**D 統一 Sans**）

| 用途 | 字體 |
|---|---|
| 中文（全頁，含標題與內文）| **Noto Sans TC**（黑體一套到底） |
| 英文 | sans-serif |

→ 跟 C 最大差異：**標題不再用 Noto Serif TC**，全部回到 Sans。襯線在實做後被認為「跟既有官網品牌個性脫節」，且容易讓標題自帶「強調感」而不像 UI 文字。

### 字級階層（D 設計師指定值）

```css
:root {
  --font-family: 'Noto Sans TC', sans-serif;

  /* font size */
  --h1-size:    39px;
  --h2-size:    30px;
  --h3-size:    24px;
  --h4-size:    19px;
  --body-size:  16px;
  --small-size: 14px;
  --note-size:  13px;

  /* font weight */
  --h1-weight:    500;
  --h2-weight:    400;
  --h3-weight:    400;
  --h4-weight:    400;
  --body-weight:  300;
  --small-weight: 300;
  --note-weight:  300;
}
```

| 級別 | 字級 | 行高 | 字重 | 用途 |
|---|---|---|---|---|
| h1 | 39px | 1.25 | **500** | 頁面主標 |
| h2 | 30px | 1.3 | **400** | Section 標題 |
| h3 | 24px | 1.4 | **400** | 子標題 |
| h4 | 19px | 1.5 | **400** | 小標題 |
| body | 16px | 1.85 | **300** | 正文（**大行高**） |
| small | 14px | 1.6 | **300** | 輔助 |
| note | 13px | 1.5 | **300** | 註解 |

→ 字重整體**偏細**：只有 h1 用 500、其他標題與正文都更輕，避免「太陽剛」、回歸文字本身的呼吸感。
→ 對比 C 的「全部 500 / body 400」，D 是「h1 500、其餘 400、內文 300」三段式階層。

### 強調大字（pull quote / 強調段落）使用規則

**觸發條件**：當段落 `font-size ≥ 22px` 且作為**獨立呈現的段落**（pull quote、promise panel、Hero h1 以外的單句強調區），即視為「強調大字段落」，必須遵守以下規則：

- **內文字數 ≤ 40 字**（含標點），超過就拆成兩段或縮小字級
- **加寬 letter-spacing**（建議 `0.04em ~ 0.08em`）跟 line-height（`1.4 ~ 1.6`），避免大字密集擠成牆
- **不要超過一個強調塊**：每個 section 最多一個大字強調段落
- **手動控制換行**：用 `<br>` 強制斷在逗點或句點後，**不可斷在詞中**（例如「不容妥協」四字必須在同一行）

→ 避免「Statement Section」失控：大字 + 長句 + 密集排版會讓內容看起來很重但其實沒講什麼，造成資訊密度過低。

### Statement Section 判斷標準（命中即違規）

回應設計回饋的明確定義。一個 section **同時命中以下 3 條以上**，即視為 Statement Section、禁用：

1. 整個 section 內**只有一句話**（沒有上下文鋪陳、沒有圖、沒有列表）
2. 該句字級 **≥ 22px**
3. 容器寬度 **≥ 880px**（撐滿大容器）
4. 上下 Section padding **≥ 96px**（大量負空間）
5. 內容是「**空洞金句**」（沒講具體事實，只是情緒性的二分法或口號式宣告）

**例外**：設計者明確標註是為了強化活動主旨（如「2026 雙 11 限定」主視覺標語）才能用，並要在文件中註記為何要破例。

### 「空洞金句」的識別

D 規範禁用的「空洞金句」舉例：

| 空洞金句樣式 | 為什麼禁用 | 改寫示範 |
|---|---|---|
| 「選擇相信，或選擇懷疑」 | 二分法情緒口號，沒講事實 | 「每一批蔬果都做質譜普篩，365 天不間斷」（具體事實） |
| 「看見的，看不見的」 | 對仗式空話 | 「每批商品檢驗 100+ 種農藥殘留」 |
| 「不只是承諾，更是…」 | 自我抬升式 | 直接寫承諾內容 |
| 「為您把關每一口」 | 過度行銷感 | 「驗出超標一律報廢，不上架」 |

→ 原則：**講具體事實、有可驗證的數字 / 動作**，而不是用抽象口號塑造氣氛。

## 六、圓角與間距

### 圓角

採用 **0px**（極簡文青風）。例外：頭像 / 圓形圖示用 50%（圓形）。

### 間距

基本單位：**4px**（強制 4px 倍數，常用值偏大）

| Token | 值 | 用途 |
|---|---|---|
| xs | 4px | 圖示與文字之間 |
| sm | 8px | 緊密區塊 |
| md | 16px | 段落內元素 |
| lg | 24px | 段落間 |
| xl | 32px | 區塊內子段落 |
| 2xl | 48px | Section 內主分隔 |
| 3xl | 64px | 中等 Section 之間 |
| 4xl | **96px** | 大 Section 之間（**常用**） |
| 5xl | **112~128px** | 主 Section padding（**桌機常用**） |

Section padding：桌機 **112~128px** / 手機 **64~80px**。

## 七、RWD 斷點與調整策略

### 三段斷點

| 斷點 | 範圍 | 對應裝置 |
|---|---|---|
| mobile | `< 768px` | 手機 |
| tablet | `768px ~ 1199px` | 平板、小桌機 |
| desktop | `>= 1200px` | 桌機、寬螢幕 |

```scss
.element { /* mobile */ }
@media (min-width: 768px) { .element { /* tablet */ } }
@media (min-width: 1200px) { .element { /* desktop */ } }
```

→ Mobile-first 寫法。

### 跨版型 RWD 共通策略

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 容器 max-width | 全寬 - 32px | 全寬 - 64px | **880px** |
| 內容左右 padding | 24px | 48px | 用 max-width |
| Section 上下 padding | 64~80px | 96px | **112~128px** |
| h1 字級 | 32px | 40px | 48px |
| h2 字級 | 26px | 32px | 36px |
| body 字級 | 16px | 17px | 18px |
| 按鈕 padding | 14px 28px | 16px 32px | 16px 32px |
| 主 nav | 漢堡選單 | 漢堡或展開 | 橫向展開 |
| 圖片裁切 | 4:5 / 4:3 | 4:5 / 4:3 | 4:5 / 16:9 / 21:9 |

## 八、整體排版策略

### 1. 頁面容器策略

| 頁面類型 | 容器策略 |
|---|---|
| 商品列表 / 詳情 | max-width **880px** 置中 |
| Blog 列表 / 文章 | max-width **720px** 置中（更窄、更精緻） |
| 訂單 / 結帳 / 表單 | max-width 524px 置中 |
| **會員中心** | **不受寬度限制（全寬）**——需要橫向空間放 sidebar + 主內容 |
| Landing / 子品牌 | 全寬，內部各區塊 max-width 880px |

### 1.1 同區塊對齊規則（明確判斷標準）

**原則**：不同 section 之間容器寬度可以不一樣（Hero 全寬、內容區 880px、表單 524px），但**同一個 section 內所有元素的容器寬度必須對齊一致**。

#### 違規 vs 合規對照

| 場景 | 違規 | 合規 |
|---|---|---|
| 同 section 內 sec-title / sec-text 限 720px，但 pillar-grid 撐 1280px | 文字短卡片寬，邊界沒對齊 | sec-title / sec-text / pillar-grid 都用同一個 container（880px 或 720px），左右邊界完全切齊 |
| Hero 全寬 + Hero 文字限 580px | OK（Hero 是 section 的特例，本來就期待全寬背景 + 內縮文字） | — |
| 加工品 section 用 21:9 全幅圖 + 加工品標題在 880px container | OK（圖是另一個視覺層，本來就跨欄） | — |

#### 實作判斷

寫一個 section 時，問自己 3 個問題：
1. 這 section 內**用了幾種容器寬度**？（理想 1 種，最多 2 種，且第 2 種必須是「視覺層」如大圖、banner）
2. 同類元素（標題、內文、卡片、按鈕）**左邊界是否完全切齊**？右邊界呢？
3. 如果不齊，差距是 **≥ 64px** 還是 **< 32px**？
   - ≥ 64px：明顯不同寬，視覺上可接受（讀者會理解為「設計刻意」）
   - < 32px：**「邊界靠近但沒對齊」，這比明顯不同還醜**，必須改

#### 常見錯誤修法

| 錯誤 | 修法 |
|---|---|
| sec-text `max-width: 720px` 跟 pillar-grid 撐 `container-wide` 1280px | 把 pillar-grid 從 container-wide 移到 container（880px），跟 sec-text 對齊 |
| 同 section 內 cta-row 跟 pillar-grid 用不同 max-width | 統一在同一個 container 內，自然對齊 |
| inline style 加了 `padding: 0 32px` 但其他元素是 `padding: 0 24px` | 拿掉 inline padding，靠 container 統一控制 |

### 2. 懸浮元素（**極簡：盡量不用**）

| 元素 | 規範 |
|---|---|
| 主 FAB | **不用**（C 文青風不適合）|
| 錨點導航 | 用標題滾動代替 |
| Sticky 底部 CTA | 僅商品詳情頁可用，主綠底白字（不用純黑）|
| 主 nav | 頂部 sticky，下捲淡出 |

### 3. 表格處理

桌機保留 table 結構；mobile 一律**卡片化**（不用橫向滾動，文青風適合卡片）。

### 4. Modal

| 屬性 | 規格 |
|---|---|
| 遮罩 | `rgba(0,0,0,0.5)` |
| 容器 | max-width 480px / mobile 全寬 - 32px |
| 圓角 | **0px** |
| Padding | 32px |
| 關閉 | X 圖示在右上 + 點遮罩可關 |

### 5. 通知 Toast

| 屬性 | 規格 |
|---|---|
| 位置 | 頁面**底部置中**（C 風格不放右上） |
| 寬度 | 桌機 480px / mobile 全寬 - 32px |
| 圓角 | **0px** |
| 動畫 | fade-up 滑入 |
| 自動消失 | 成功 3 秒 / 錯誤手動關閉 |

### 6. 圖文搭配（Media + Text）

用於：產品介紹、品牌故事、產地報導、Landing Page 中段。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 主 layout | 上下堆疊（圖→文）| 上下堆疊 | **左圖右文 60/40 或 40/60**（C 不對稱、留白多）|
| 圖片比例 | 4:3 或 4:5 | 4:5 | **4:5 直立**（呼應商品卡）|
| 圖文間距 | 24px | 32px | **64~96px**（中間 gap，比一般大）|
| 文字 padding | 24px | 32px | 48px |
| 多段交替 | 全部圖在上 | 全部圖在上 | 左圖右文 / 右圖左文交替 |

→ **圓角 0px**；標題 h3 28px Serif；內文 body 17~18px；圖片用 `<figure>` + `<figcaption>` 寫圖說（Serif italic、灰字）。

#### 6.1 適用門檻（避免圖文比例失衡）

4:5 直立圖在桌機高度 = 容器寬度 × 1.25，比想像中高。若右欄只放「標題 + 兩行 intro」，圖把版面撐開後右側留白會異常醜。

| 右欄內容量 | 建議版型 |
|---|---|
| **多**（≥ 1 個 sub-standard 或 ≥ 5 段內容） | **4:5 split-block**（標準圖文搭配，圖文等高自然平衡） |
| **少**（標題 + intro + 短列表，≤ 3 段內容） | **改用全幅 21:9 橫幅 + 集中窄欄**（不硬塞 split） |

→ 內容單薄時硬用 4:5 split 會被讀作「為了用版型而用」，反而破壞文青質感。

#### 6.2 Sticky 圖片黏滯規則

當右欄內容很長（捲動超過一個圖高度），桌機可讓圖 sticky 黏在欄位內、跟著文字滾動，避免「圖看完就消失」。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| `.split-image` position | **`static`**（必須關掉 sticky）| static | **`sticky` / top: 80px** |

→ **手機版必須關 sticky**：split-block 在 `< 900px` 變單欄上下堆疊，sticky 會讓圖在滾動時黏住、疊到下方文字上，造成字疊圖。
→ CSS 範例：

```scss
.split-image {
  position: sticky;
  top: 80px;
}
@media (max-width: 900px) {
  .split-image {
    position: static;
    top: auto;
  }
}
```

### 7. 圖片段落（Image Block）

用於：產地實景、檢驗實驗室、團隊照、產品大圖、品牌氛圍圖。

| 形式 | mobile | tablet | desktop |
|---|---|---|---|
| 單張大圖（feature）| **全寬 21:9**（C 加寬） | 全寬 21:9 | **全寬 21:9**（不限 1140px）|
| 雙圖並排 | 上下堆疊 | 雙欄 4:5 | 雙欄 4:5 |
| 三圖網格 | 上下堆疊 | 雙欄 + 一 | 三欄 4:5 |
| 四圖網格 | 雙欄 1:1 | 雙欄 1:1 | 四欄 1:1 |
| 圖片 gap | 12~16px | 24px | **48px**（C 大留白）|
| 圖說 caption | 圖下 16px、`note 13px`、Serif、灰字 | 同 | 同 |

→ **圓角 0px**；用 `<figure>` + `<figcaption>` 包裝；圖片 alt 必填。

### 8. 圖片按鈕組（Image Card Buttons）

用於：分類入口、子品牌入口、活動入口、首頁 CTA 區。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 卡數 | 1（縱列）| 2 | 3~4 |
| 圖片比例 | 4:5 | 4:5 | **4:5 直立**（C 直立式）|
| 文字位置 | 圖下 | 圖下 | **圖下卡片式為主**（C 不推 overlay）|
| Hover | 圖片淡淡淡入暗（不 zoom）| 同 | 同 |
| 字型 | Sans 標題 + Serif italic 副標 | 同 | 同 |

C 推薦變體：

**圖下卡片式**（C 唯一推薦，AEO 友善）
- 圖（4:5 直立）→ 標題（h4 22px Sans Medium）→ 副標（small 15px Serif italic 灰字）→ CTA「了解更多 →」（文字底線型）

→ 整張卡 `<a>` 包裹，整張可點擊；hover 圖片變暗 5%（不 zoom）。

## 十四、AI 友善網站（SEO / AEO）

讓 AI 搜尋引擎（ChatGPT、Perplexity、Gemini 等）能正確理解、引用網站內容。原則：**結構清楚 + 內容明確 + 機器可讀**。

C 方案**天然符合** AI 預設框架（大留白、Serif 標題、單欄堆疊）。

### 1. 語意化 HTML 結構

| 規則 | 說明 |
|---|---|
| 每頁只有一個 `<h1>` | 且包含核心關鍵字（商品名 / 類別名 / 品牌名）|
| H2~H6 嚴格層級遞進 | 不跳級（不能 H1 → H3）；不為視覺用 heading |
| 區塊用 semantic tag | `<article>` / `<section>` / `<nav>` / `<main>` / `<header>` / `<footer>` / `<aside>` |
| 列表用 `<ul>` / `<ol>` | 不用 `<div>` 模擬列表 |
| 表格用 `<table>` | `<thead>` `<tbody>` 完整；不用 div grid 假表格 |
| 引述用 `<blockquote>` | 不用單純 italic / 引號 |

### 2. Heading 內容原則

- **H1**：頁面主標題，包含核心關鍵字
- **H2**：主要區塊標題
- **H3**：子區塊
- 視覺需要大字但語意上不是標題 → 用 `<p class="text-large">`，**不要用 heading 標籤**

### 3. 內容結構（AEO 友善）

| 原則 | 做法 |
|---|---|
| 問答式 | 用問題當 H2/H3，下方接答案 |
| 倒金字塔（結論先行） | 段落第一句寫結論，後面補充細節 |
| 用 list / table 結構化 | 重點資訊用 `<ul>` 或 `<table>`，AI 容易解析 |
| 具體數據 | 用具體數字、單位、日期 |
| 避免空洞行銷 | 不寫「最好的」「業界第一」「革命性」 |

### 4. Metadata 與結構化資料

| 項目 | 規則 |
|---|---|
| `<title>` | 60 字內，包含主關鍵字 + 品牌名 |
| `<meta name="description">` | 155 字內，自然描述頁面，不堆關鍵字 |
| Open Graph | og:title / og:description / og:image / og:url |
| **Schema.org JSON-LD** | 商品 → `Product`；文章 → `Article`；組織 → `Organization`；FAQ → `FAQPage`；麵包屑 → `BreadcrumbList` |

### 5. 圖片 alt 文字

- 內容圖：alt 描述圖片內容
- 商品圖：alt 含商品名 + 主特徵
- 裝飾圖：`alt=""`（空字串）
- 避免：「圖片」「banner」「圖 1」這種無意義 alt
- 避免堆關鍵字

### 6. URL 與內部連結

- URL 簡短、語意清楚
- 內部連結 anchor text 描述性（避免「點這裡」）
- 麵包屑導覽 + Schema BreadcrumbList

### 7. AI Crawler 設定

| 項目 | 規則 |
|---|---|
| robots.txt | 開放主流 AI bot：`GPTBot` / `ClaudeBot` / `PerplexityBot` / `Google-Extended` / `CCBot` |
| sitemap.xml | 完整列出所有公開頁，每月更新 |
| SSR / 靜態化 | **重要內容直接出現在 HTML**，不等 JS 渲染 |
| 頁面載入速度 | 3 秒內首屏可見 |

### 核心原則三條

1. **結構清楚**：H1/H2/H3 階層 + semantic HTML
2. **內容明確**：問答式 + 倒金字塔 + 具體數據
3. **機器可讀**：Schema.org + alt + 不藏在 JS

C 是「**換包裝**」（主色不變、版型字型重做）：

## 二十、設計收斂規則（D 比 C 嚴的部分）

設計實做後累積的回饋，整理成 12 條收斂規則。這些都是「**C 容許但 D 禁用**」的具體場景。

### 引導與點擊行為

1. **Hero 內不放 CTA 按鈕**——避免使用者一進站還沒看內容就被誘導跳轉，說服過程不夠就跳走轉換率反而差
2. **任何標題不准做成像按鈕的視覺**（無背景色塊、邊框、Pill 形狀），怕被誤認為可點
3. **沒有連結或互動行為的卡片/區塊一律不加 hover 效果**（不變色、不浮起、不變邊框、不放大）

### 對齊與容器

4. **不同段落的容器寬度可以不一樣**，但**同一個區塊內**對齊規則必須一致——例：純文字段落接圖片時，文字左邊跟圖左邊要對齊；左右並排時兩欄寬度比例固定
5. **避免「邊界靠近但沒對齊」**的情況，這比「明顯不同寬」還醜

### 裝飾與點綴

6. **不要把內文中的單一字當裝飾字**（例如「無毒農的『**好**』在哪」拉出來放大），必要時改用 icon 圖示或數字編號
7. **圖片下方預設不加說明字**（除非設計特別指定有重要 context 要交代）

### 顏色與背景

8. **不要整片深綠背景 + 白色文字**（C 的 promise panel 那種戲劇化深森林綠 panel，D 砍掉），淺灰或淺綠 OK

### 排版節奏

9. **強調用大字體段落**：內文 ≤ 40 字，且 letter-spacing + line-height 都加寬，避免閱讀負擔
10. **避免 Statement Section**（滿版單句金句區塊）——除非設計者明確標註是為了強化活動主旨而刻意強化，否則一律不用。理由：資訊密度過低但其實沒講什麼重要內容
11. **標題換行特別留意**——理想是不換行，必須換行時要在**逗點或合適分段處**換，不能斷在詞中

### RWD

12. **手機版按鈕需置中**，至少要與其他元素邊界對齊；不可漂浮在欄位邊緣

### Banner Slider 修訂

13. **Banner slider 不放左右切換按鈕**——下方圓點 + 自動換頁就夠用。左右箭頭壓在圖上反而會蓋到圖片中的關鍵元素或文字

→ 任何違反以上 13 條的版型，**都不能直接使用，必須經 Owner 同意才能例外**。

## 二十一、不可用版型（明確禁止）

D 風格的精神是「**減法 × 不誤導**」，不確定要不要留的元素先砍。

### 視覺類禁用

1. **滿版彩色漸層 banner**（紫到藍、彩虹漸層等套版方向）
2. **圓角卡片 + 彩色左邊框 + icon + 標題 + 描述四件組**（一眼 AI Slop 模板）
3. **任何 emoji 裝飾**（🚀✨💡 當項目符號或標題開頭）
4. **線條風格 SVG 插圖**（機器人、燈泡、齒輪等抽象示意）
5. **fill icon**（D 用 thin line）
6. **假數據過度整齊**（`$9,999`、`99.9% uptime`、`1,234 users` 這種 demo 味）
7. **整片深綠 panel + 白字**（D 取消這個 pattern）

### 結構類禁用

8. **多欄密集網格**（D 用單欄堆疊；列表頁最多 2 欄）
9. **彩色情境色濫用**（D 以主綠 + 中性色為主，情境色節制）
10. **不規則比例的圖片裁切**（4:3 / 16:9 / 4:5 / 21:9 / 1:1 以外的奇怪比例）
11. **過度裝飾邊框**（雙線、虛線、漸層邊框）
12. **浮動 Tooltip**（除既有 hover 簡述外的彈出框）
13. **5px、7px、10px 等非 4px 倍數的間距**
14. **4px、6px、8px 圓角**（D 統一 0px）
15. **FAB / 過多懸浮按鈕**（極簡）
16. **圖片按鈕 overlay 變體**（文字壓圖上影響閱讀，D 只用「圖下卡片式」）
17. **Banner slider 左右切換箭頭**（D 只用下方圓點 + 自動換頁）
18. **Hero 區 CTA 按鈕**
19. **Statement Section**（除非明確標註強化主旨）
20. **無連結的卡片/區塊 hover 效果**
21. **按鈕樣式的標題**
22. **內文單字當裝飾字**

### 文案類禁用

23. **空洞行業術語**（「賦能」「閉環」「抓手」「下沉」）
24. **過度稱讚式文案**（「完美的」「極致的」「革命性的」）
25. **過長標題 / 段落**（標題 ≤ 20 字、段落 ≤ 4 行）
26. **大字強調段落 > 40 字**（font-size ≥ 22px 即觸發 ≤ 40 字限制）
27. **二分法行銷金句**（「選擇 X 或 Y」「看見/看不見」「不只是 X 更是 Y」這類對仗式情緒口號，沒講具體事實）
28. **標題或大字段落斷在詞中換行**（例如「不容\n妥協」、「品質\n保證」這種把詞拆兩行；必須用 `<br>` 強制斷在逗點/句點後）

→ 不在「常用版型」清單內、且未經 Owner 同意的版型，都視為不可用。
