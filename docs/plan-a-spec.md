## 四、字級階梯

### 字體

| 用途 | 字體 | 適用範圍 |
|---|---|---|
| 中文（主站、粥寶寶、綠拿鐵、等家寶寶） | **Noto Sans TC** | 全站主要 |
| 中文（益菓保） | （待 Owner 確認；舊版為 GenSenRounded） | 子品牌特例 |
| 英文 | sans-serif（系統預設） | 全站 |

### 字級階層（自 D 同步：字級基準 39/30/24/19、字重整體偏細、body 大行高）

```css
:root {
  --font-family: 'Noto Sans TC', sans-serif;

  /* font size */
  --h1-size:    39px;
  --h2-size:    30px;
  --h3-size:    24px;
  --h4-size:    19px;
  --body-size:  16px;
  --small-size: 14px;
  --note-size:  13px;

  /* font weight */
  --h1-weight:    500;
  --h2-weight:    400;
  --h3-weight:    400;
  --h4-weight:    400;
  --body-weight:  300;
  --small-weight: 300;
  --note-weight:  300;
}
```

| 級別 | 字級 | 行高 | 字重 | 用途 |
|---|---|---|---|---|
| h1 | 39px | 1.25 | 500 | 頁面主標題 |
| h2 | 30px | 1.3 | 400 | Section 標題 |
| h3 | 24px | 1.4 | 400 | 子標題 |
| h4 | 19px | 1.5 | 400 | 小標題 |
| body | 16px | 1.85 | 300 | 正文（大行高）|
| small | 14px | 1.6 | 300 | 輔助文字 |
| note | 13px | 1.5 | 300 | 灰字註解 |

→ 字重整體**偏細**：只有 h1 用 500、其他標題與正文都更輕，回歸文字本身的呼吸感。RWD 各斷點字級見「七、RWD」。

### 強調大字（pull quote / 強調段落）使用規則

**觸發條件**：當段落 `font-size ≥ 22px` 且作為**獨立呈現的段落**（pull quote、強調區、Hero h1 以外的單句強調區），即視為「強調大字段落」，必須遵守：

- **內文字數 ≤ 40 字**（含標點），超過就拆兩段或縮小字級
- **加寬 letter-spacing**（建議 `0.04em ~ 0.08em`）與 line-height（`1.4 ~ 1.6`），避免大字密集擠成牆
- **每個 section 最多一個**大字強調段落
- **手動控制換行**：用 `<br>` 強制斷在逗點或句點後，**不可斷在詞中**（例如「不容妥協」四字必須在同一行）

### Statement Section 判斷標準（命中即違規）

一個 section **同時命中以下 3 條以上**，即視為 Statement Section、禁用：

1. 整個 section 內**只有一句話**（沒有上下文鋪陳、沒有圖、沒有列表）
2. 該句字級 **≥ 22px**
3. 容器寬度 **≥ 880px**（撐滿大容器）
4. 上下 Section padding **≥ 96px**（大量負空間）
5. 內容是「**空洞金句**」（沒講具體事實，只是情緒性二分法或口號式宣告）

**例外**：設計者明確標註是為了強化活動主旨（如「2026 雙 11 限定」主視覺標語）才能用，並在文件中註記破例原因。

### 「空洞金句」的識別

| 空洞金句樣式 | 為什麼禁用 | 改寫示範 |
|---|---|---|
| 「選擇相信，或選擇懷疑」 | 二分法情緒口號，沒講事實 | 「每一批蔬果都做質譜普篩，365 天不間斷」|
| 「看見的，看不見的」 | 對仗式空話 | 「每批商品檢驗 100+ 種農藥殘留」|
| 「不只是承諾，更是…」 | 自我抬升式 | 直接寫承諾內容 |
| 「為您把關每一口」 | 過度行銷感 | 「驗出超標一律報廢，不上架」|

→ 原則：**講具體事實、有可驗證的數字 / 動作**，而不是用抽象口號塑造氣氛。

## 七、RWD 斷點與調整策略

### 三段斷點

| 斷點 | 範圍 | 對應裝置 |
|---|---|---|
| mobile | `< 768px` | 手機 |
| tablet | `768px ~ 1199px` | 平板、小桌機 |
| desktop | `>= 1200px` | 桌機、寬螢幕 |

```scss
.element { /* mobile */ }
@media (min-width: 768px) { .element { /* tablet */ } }
@media (min-width: 1200px) { .element { /* desktop */ } }
```

→ Mobile-first 寫法。SCSS 現有 550 / 580 / 941 等斷點 sweep 為三段。

### 跨版型 RWD 共通策略

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 容器 max-width | 全寬 - 32px | 全寬 - 64px | **880px**（自 D 同步，原 1140px）|
| 內容左右 padding | 16px | 32px | 用 max-width 收 |
| Section 上下 padding | 32px | 48px | 64px |
| h1 字級 | 32px | 40px | 48px |
| h2 字級 | 26px | 32px | 36px |
| body 字級 | 16px | 17px | 18px |
| 按鈕 padding | 10px 20px | 12px 24px | 12px 28px |
| 主 nav | 漢堡選單 | 漢堡或展開 | 橫向展開 |
| 圖片裁切 | 4:5 / 4:3 | 4:5 / 4:3 | 4:5 / 16:9 / 21:9 |

→ 各版型在此基礎上加自身欄數 / 排列調整，見下節。
→ 字級基準同步 D（h1 39 / body 16 為基準值，RWD 由上表逐斷點放大）。

### 手機版按鈕對齊（自 D 同步）

手機版按鈕需**置中**，至少要與其他元素的邊界對齊，不可漂浮在欄位邊緣。全寬按鈕除外（本來就靠左右 padding 對齊）。

| **會員中心 / 後台型功能頁** | **不受寬度限制（全寬）** | 需要橫向空間放 sidebar + 主內容 |
| 首頁 / 子品牌 LandingPage | 全寬，內部各區塊各自有容器 | Hero 區塊全寬，內容區塊用 880px |

### 1.1 同區塊對齊規則（自 D 同步）

**原則**：不同 section 之間容器寬度可以不一樣（Hero 全寬、內容區 880px、表單 524px），但**同一個 section 內所有元素的容器寬度必須對齊一致**。

#### 違規 vs 合規對照

| 場景 | 違規 | 合規 |
|---|---|---|
| 同 section 內標題 / 內文限 720px，但卡片網格撐 1140px | 文字短卡片寬，邊界沒對齊 | 標題 / 內文 / 卡片網格都用同一個 container（880px），左右邊界完全切齊 |
| Hero 全寬 + Hero 文字限 580px | OK（Hero 是 section 特例，期待全寬背景 + 內縮文字）| — |
| 某 section 用 21:9 全幅圖 + 標題在 880px container | OK（圖是另一個視覺層，本來就跨欄）| — |

#### 實作判斷（寫一個 section 時自問 3 題）

1. 這 section 內**用了幾種容器寬度**？（理想 1 種，最多 2 種，且第 2 種必須是「視覺層」如大圖、banner）
2. 同類元素（標題、內文、卡片、按鈕）**左邊界是否完全切齊**？右邊界呢？
3. 如果不齊，差距是 **≥ 64px** 還是 **< 32px**？
   - ≥ 64px：明顯不同寬，視覺上可接受（讀者會理解為「設計刻意」）
   - < 32px：**「邊界靠近但沒對齊」，這比明顯不同還醜**，必須改

#### 常見錯誤修法

| 錯誤 | 修法 |
|---|---|
| 內文 `max-width: 720px` 跟卡片網格撐 1140px | 把卡片網格移到 880px container，跟內文對齊 |
| 同 section 內按鈕列跟卡片網格用不同 max-width | 統一在同一個 container 內，自然對齊 |
| inline style 加了 `padding: 0 32px` 但其他元素是 `padding: 0 24px` | 拿掉 inline padding，靠 container 統一控制 |

### 2. 懸浮元素（FAB / 錨點 / Sticky CTA）

| 元素 | 位置 | 規格 |
|---|---|---|
| 主 FAB（電話訂購、LINE、回頂部） | 右下角 | 距右 20px、距下 20px；圓形 48×48px；多顆垂直排列、gap 12px |
| 錨點導航（長頁面 section 跳轉） | 桌機：左側 sticky；手機：頂部 sticky 橫向滾動 | 桌機寬 200px、距左 32px |
| Sticky 底部 CTA（mobile only） | 畫面底部 | 高 64px、白底 + 上邊陰影；用於詳情頁的「加入購物車」「立即捐款」 |
| 主 nav | 頂部 sticky | 下捲時隱藏、上捲時顯示 |

### 3. 表格處理（自 D 同步：mobile 一律卡片化）

桌機保留完整 table 結構；mobile **一律卡片化**——每 row 變獨立卡片，用 label/value 對照，不用橫向滾動。

| 屬性 | 做法 |

| 桌機 | 完整 `<table>`（`<thead>` `<tbody>` 結構不變）|
| mobile | 每 row → 一張卡片，欄名當 label、值當 value 上下排 |

### 5. Banner（Slider）

用於：首頁 Hero、商品列表 Hero、活動主視覺、子品牌 Hero、會員專區頂部

#### 現況（用 Playwright 抓首頁 banner 校對）

| 屬性 | mobile（現況） | desktop（現況） |
|---|---|---|
| 比例 | ~1:1（方形 0.98） | ~2.19:1（≈21:9 扁形） |
| 容器寬度 | 330px（含 padding） | 1040px |
| 左右箭頭 | 顯示 | 顯示 |
| 下方圓點 | 顯示（8 個） | 顯示（8 個） |
| 自動播放間隔 | 3 秒（slick `autoplaySpeed: 3000`） | 3 秒 |
| 切換動畫時間 | 1000ms | 1000ms |
| 切換方式 | swipe + 箭頭 + 圓點 | swipe + 箭頭 + 圓點 |
| 套件 | slick carousel | slick carousel |

→ **發現的問題**：
- 桌機 / mobile 比例差距很大（21:9 vs 1:1），視覺節奏不統一
- autoplay 3 秒對 banner 上有大量文字的情境偏快

#### 未來規範

| 屬性 | mobile | desktop |
|---|---|---|
| **比例** | **維持 1:1 方形**（不強制 16:9） | **統一 16:9**（從 ~21:9 改） |
| 容器寬度 | 全寬（含左右 padding） | 全寬 或 1140px |
| **左右箭頭** | **不放**（自 D 同步移除）| **不放**（自 D 同步移除）|
| **圓點位置** | **圖片外下方獨立區域**（不疊在圖片上） | 同 |
| 圓點數量 | ≤ 8 個 | ≤ 8 個 |
| 自動播放間隔 | **7 秒**（自 D 同步）| **7 秒** |
| 切換動畫時間 | 300~500ms | 300~500ms |
| 切換方式 | swipe + 圓點 + 自動換頁 | swipe + 圓點 + 自動換頁 |

→ **桌機**改 16:9（解決現況 ~21:9 太扁的問題）；**mobile 維持 1:1 方形**（不統一比例）。
→ **Banner 不壓左右切換箭頭**（自 D 同步）：下方圓點 + 定時自動換頁就夠用；箭頭壓在圖上容易蓋到圖中的關鍵元素或文字。
→ **圓點放圖片外下方**獨立區域，不疊在圖片上（避免遮蓋圖片內容）。
→ 自動播放間隔放慢到 **7 秒**（拿掉箭頭後更依賴自動換頁，慢一點較好讀）。
→ 文字位置、CTA 位置等**圖內排版** DS 不規範（避免限制設計）。
→ 套件維持 slick 或評估升級到更輕量方案。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| 主 layout | 上下堆疊（圖→文） | 上下堆疊 | **左圖右文 50/50**（或反向） |
| 圖片比例 | 4:3 | 4:3 或 16:9 | 4:3 或 16:9 |
| 圖片寬度 | 100% | 100% | 50% |
| 文字 padding | 16px | 24px | 32~48px |
| 圖文間距 | 16px（圖下） | 24px | 48px（中間 gap） |
| 段落順序 | 圖 → 標題 → 內文 → CTA | 同 | 並列 |
| 多段交替 | 全部圖在上 | 全部圖在上 | 左圖右文 / 右圖左文交替 |

→ 圓角 4px；標題 h3 24px；內文 body 16px。
→ **圖片下方預設不加說明字（figcaption）**（自 D 同步），除非設計特別指定有重要 context 要交代。圖片仍用 `<figure>` 包裝、必填 `alt` 描述（alt 與 caption 不同，alt 仍是 AEO/無障礙必需）。

#### 6.1 Sticky 圖片黏滯規則（自 D 同步）

當右欄內容很長（捲動超過一個圖高度），桌機可讓圖 sticky 黏在欄位內、跟著文字滾動，避免「圖看完就消失」。

| 屬性 | mobile | tablet | desktop |
|---|---|---|---|
| `.split-image` position | **`static`**（必須關掉 sticky）| static | **`sticky` / top: 80px** |

→ **手機版必須關 sticky**：split-block 在 `< 900px` 變單欄上下堆疊，sticky 會讓圖在滾動時黏住、疊到下方文字上，造成字疊圖。

```scss
.split-image {
  position: sticky;
  top: 80px;
}
@media (max-width: 900px) {
  .split-image {
    position: static;
    top: auto;
  }
}
```

## 十四、AI 友善網站（SEO / AEO）

讓 AI 搜尋引擎（ChatGPT、Perplexity、Gemini 等）能正確理解、引用網站內容。原則：**結構清楚 + 內容明確 + 機器可讀**。

### 1. 語意化 HTML 結構

| 規則 | 說明 |
|---|---|
| 每頁只有一個 `<h1>` | 且包含核心關鍵字（商品名 / 類別名 / 品牌名） |
| H2~H6 嚴格層級遞進 | 不跳級（不能 H1 → H3）；不為視覺用 heading |
| 區塊用 semantic tag | `<article>` / `<section>` / `<nav>` / `<main>` / `<header>` / `<footer>` / `<aside>` |
| 列表用 `<ul>` / `<ol>` | 不用 `<div>` 模擬列表 |
| 表格用 `<table>` | `<thead>` `<tbody>` 完整；不用 div grid 假表格 |
| 引述用 `<blockquote>` | 不用單純 italic / 引號 |

### 2. Heading 內容原則

- **H1**：頁面主標題，包含核心關鍵字（如「無毒農粥寶寶 - 安心副食品」）
- **H2**：主要區塊標題（如「商品介紹」「規格」「安全檢驗」）
- **H3**：子區塊（如「分階段商品」「使用方式」）
- 視覺需要大字但語意上不是標題 → 用 `<p class="text-large">`，**不要用 heading 標籤**

### 3. 內容結構（AEO 友善）

| 原則 | 做法 | 例子 |
|---|---|---|
| 問答式 | 用問題當 H2/H3，下方接答案 | H2「無毒農跟有機認證有什麼差別？」+ 段落答案 |
| 倒金字塔（結論先行） | 段落第一句寫結論，後面補充細節 | 「我們每批檢驗 365 天。為了讓...」 |
| 用 list / table 結構化 | 重點資訊用 `<ul>` 或 `<table>`，AI 容易解析 | 商品規格表、檢驗項目清單 |
| 具體數據 | 用具體數字、單位、日期 | 「殘留量 ≤ 國家標準的 1/2」「365 天每日篩檢」 |
| 避免空洞行銷 | 不寫「最好的」「業界第一」「革命性」 | 改寫成「全台唯一千萬等級質譜實驗室」（具體事實） |

### 4. Metadata 與結構化資料

| 項目 | 規則 |
|---|---|
| `<title>` | 60 字內，包含主關鍵字 + 品牌名（如「益菓DHA魚油｜無毒農益菓保」） |
| `<meta name="description">` | 155 字內，自然描述頁面，不堆關鍵字 |
| Open Graph | og:title / og:description / og:image / og:url（社群分享預覽用） |
| **Schema.org JSON-LD** | 商品頁 → `Product`；文章 → `Article`；組織 → `Organization`；FAQ → `FAQPage`；麵包屑 → `BreadcrumbList` |

### 5. 圖片 alt 文字

- 內容圖：alt 描述圖片內容（如「金黃奇異果切片，露出黃色果肉」）
- 商品圖：alt 含商品名 + 主特徵（如「無毒農粥寶寶 八口味米糊組合」）
- 裝飾圖：`alt=""`（空字串，避免 AI 干擾）
- 避免：「圖片」「banner」「圖 1」這種無意義 alt
- 避免堆關鍵字（如「奇異果 紐西蘭 黃金 進口 水果」）

### 6. URL 與內部連結

- URL 簡短、語意清楚：`/Products/Kitchenette` ✓ vs `/p?id=12345` ✗
- 內部連結 anchor text 描述性：「無毒農粥寶寶系列」 ✓ vs 「點這裡」 ✗
- 麵包屑導覽 + Schema BreadcrumbList

### 7. AI Crawler 設定

| 項目 | 規則 |
|---|---|
| robots.txt | 開放主流 AI bot：`GPTBot` / `ClaudeBot` / `PerplexityBot` / `Google-Extended` / `CCBot` |
| sitemap.xml | 完整列出所有公開頁，每月更新 |
| SSR / 靜態化 | **重要內容直接出現在 HTML**，不等 JS 渲染（AI 多半不執行 JS） |
| 頁面載入速度 | 3 秒內首屏可見（影響 AI 抓取頻率） |

### 核心原則三條

1. **結構清楚**：H1/H2/H3 階層 + semantic HTML
2. **內容明確**：問答式 + 倒金字塔 + 具體數據
3. **機器可讀**：Schema.org + alt + 不藏在 JS

- 若 B/C 加動畫，原則：
  - Duration：150~300ms（短暫不拖延）
  - Easing：`ease-out`（自然減速）

→ A 階段：**沿用既有 slider 行為**，不新增動畫。

## 二十、不可用版型（A 方案明確禁止）

> 此段為文件最後，含視覺案例見 [方案A-預覽.html](方案A-預覽.html) 對應段落。

### 視覺類禁用

1. **滿版彩色漸層 banner**（紫到藍、彩虹漸層等套版方向）
2. **圓角卡片 + 彩色左邊框 + icon + 標題 + 描述四件組**（一眼 AI Slop 模板）
3. **Emoji 當 UI 裝飾**（🚀✨💡 當項目符號或標題開頭）
4. **線條風格 SVG 插圖**（機器人、燈泡、齒輪等抽象示意）
5. **假數據過度整齊**（`$9,999`、`99.9% uptime`、`1,234 users` 這種 demo 味）
6. **整片深綠背景 + 白色文字**（戲劇化深綠 panel；自 D 同步，淺灰 / 淺綠 OK）

### 結構類禁用

7. **不規則比例的圖片裁切**（4:5 / 4:3 / 16:9 / 21:9 / 1:1 以外的奇怪比例）
8. **過度裝飾邊框**（雙線、虛線、漸層邊框）
9. **浮動 Tooltip**（除既有 hover 簡述外的彈出框）
10. **5px、7px、10px 等非 4px 倍數的間距**
11. **5px、6px、8px 圓角**（A 統一 4px）

### 引導與點擊類禁用（自 D 同步）

12. **Hero 區內放 CTA 按鈕**（避免未看內容就被誘導跳轉；CTA 放內容區塊後）
13. **任何標題做成按鈕樣式**（背景色塊 / 邊框 / Pill 形狀，怕被誤點）
14. **沒有連結或互動行為的卡片 / 區塊加 hover 效果**（不變色、不浮起、不變邊框、不放大）
15. **把內文中的單一字當裝飾字**（單字拉大當裝飾；必要時改用 icon 或數字編號）

### 排版與文案類禁用

16. **空洞行業術語**（「賦能」「閉環」「抓手」「下沉」）
17. **過度稱讚式文案**（「完美的」「極致的」「革命性的」）
18. **Statement Section**（滿版單句金句區，命中判斷標準見「四、字級階梯」；除非明確標註強化活動主旨）（自 D 同步）
19. **二分法行銷金句**（「選擇 X 或 Y」「看見/看不見」「不只是 X 更是 Y」這類對仗式情緒口號，沒講具體事實）（自 D 同步）
20. **大字強調段落 > 40 字**（font-size ≥ 22px 即觸發 ≤ 40 字限制）（自 D 同步）
21. **標題或大字段落斷在詞中換行**（如「不容\n妥協」「品質\n保證」；必須用 `<br>` 強制斷在逗點 / 句點後）（自 D 同步）
