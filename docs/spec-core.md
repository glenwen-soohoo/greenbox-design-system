# DesignSystem 核心規範（護欄 + 品牌 token）

> 建立日期：2026-07-13
> 類型：參考資料
> 需求 Owner：glen.wen
> 適用對象：所有製作無毒農網頁的人與 AI agent（形象頁、活動頁、日常頁面一律適用）
> 資料來源：由「方案 A（形象頁現況規範）」與「方案 D（活動頁精煉規範）」去重合併，抽出兩者共通的護欄與品牌 token
> 演進來源：原本形象頁用 A、活動頁用 D 兩份並行、重複性高。改為「一份核心 + 形象頁延伸 + 活動頁延伸」三層架構，本檔為核心。

---

## 這份文件的定位

這是**地基**。無論做哪一種頁面，這裡的規則都要遵守。它只放「不管長什麼樣都不能破的東西」——**不決定頁面的風格長相**，長相由兩份延伸決定：

- **[形象頁延伸](4-DesignSystem-形象頁.md)**：無毒農品牌門面的固定長相（低頻更新）。
- **[活動頁延伸](4-DesignSystem-活動頁.md)**：少數風格建議 + 每檔外掛的風格參考素材（高頻新建）。

### 分級用語（給人也給 AI 讀）

本系列文件所有規則都標成三級，AI agent 套用時依此判斷能不能改：

| 級別 | 意義 | 能不能違背 |
|---|---|---|
| **MUST（護欄）** | 硬約束 | **不可**。就算客戶要求、活動主題素材長那樣，也要擋下並說明 |
| **SHOULD（建議）** | 預設傾向 | 可，但要有理由（通常是活動主題參考帶來的覆蓋） |
| **MAY（自由）** | 開放 | 隨頁面 / 主題自由決定 |

→ **本核心文件內的規則，除非特別註記，全部是 MUST。** SHOULD / MAY 主要出現在兩份延伸裡。

### 1.4 字型（MUST）

| 用途 | 字型 |
|---|---|

| 英文 | sans-serif |

### 1.5 字級階梯（MUST）

```css
:root {
  --font-family: 'Noto Sans TC', sans-serif;
  /* font size */
  --h1-size: 48px;  --h2-size: 39px;  --h3-size: 30px;  --h4-size: 24px;
  --body-size: 19px; --small-size: 16px; --note-size: 14px;
  /* font weight */
  --h1-weight: 500; --h2-weight: 500; --h3-weight: 400; --h4-weight: 400;
  --body-weight: 400; --small-weight: 400; --note-weight: 300;
}
```

| 級別 | 字級 | 行高 | 字重 | 用途 |
|---|---|---|---|---|
| h1 | 48px | 1.25 | 500 | 頁面主標 |
| h2 | 39px | 1.3 | 500 | Section 標題 |
| h3 | 30px | 1.4 | 400 | 子標題 |
| h4 | 24px | 1.5 | 400 | 小標題 |
| body | 19px | 1.85 | 400 | 正文（大行高）|
| small | 16px | 1.6 | 400 | 輔助（特殊情境）|
| note | 14px | 1.5 | 300 | 最小級（圖說 / footer）|

→ 本階梯由原版整體**往上調一級**（含字級與字重）：最小級 note 由 13/300 提為 14/300，其餘各級繼承原本上一階的值，h1 補到 48/500。行高服務用途、不平移。
→ 字重分佈：note 300、small / body / h3 / h4 400、h1 / h2 500。RWD 各斷點放大值見 §三。
→ 活動頁 h1 字級 **MAY** 依主視覺放大，但字型與字重階層關係不變。

### 1.5.1 字級使用紀律（MUST）

實測常見「用錯層級」，一律避免：

- **內文一律用 body**；只有特殊情境（表格附註、標籤、次要輔助）才用 small，**不可濫用 small 當內文**。
- **禁止拿 body 當標題**（標題一律用 h1~h4）；**禁止拿 note / small 當內文**。這兩種是最常見的誤用。
- **note 是最小級，僅用於圖片說明文字、footer、版權宣告等邊緣資訊**，不是內文層級。
- **活動頁預設不使用 note 層級**（活動頁少有需要這種邊緣小字的情境）。

### 1.6 品牌識別規則（什麼不可動、什麼可依主題調）

| 項目 | 規則 | 級別 |
|---|---|---|
| Logo | **不可改色、不可變形** | MUST |
| 主字型 Noto Sans TC | **不可換** | MUST |
| 子品牌主色 | 做該子品牌頁時**不可換** | MUST |
| 品牌主綠 `#4ba83b` / 標題深綠 `#008000` | 品牌識別主色。形象頁鎖定使用；**活動頁建議優先採用，但主色與 CTA 色可依主題參考改用主題色** | SHOULD |
| 主題色 / 主題輔色 | 活動頁 **MAY** 依主題參考設定主色、輔色與 CTA 色（節慶紅金、聯名色等）；仍受無障礙對比（§五）與 AI Slop（§七）約束 | MAY |

→ 品牌綠不再是「每頁 CTA 都必須」的硬規定——logo 已提供品牌識別，活動頁可為了主題氣氛改用主題色 CTA。形象頁仍鎖定品牌綠。
→ 需要既有色票裡沒有的功能色（warning / info 等）時，**先問 Owner**，不可自己湊 hex。

## 二、不誤導點擊（MUST）

保護使用者不被視覺誤導，這組在任何頁面、任何主題下都成立。

1. **Hero 內不放 CTA 按鈕**——避免使用者還沒看內容就被誘導跳轉；CTA 放在內容區塊之後。
2. **任何標題、裝飾文字不可做成按鈕樣式**（不加背景色塊、邊框、Pill 膠囊形狀），怕被誤認為可點。需要強調用字級 / 字重 / 顏色。
3. **沒有連結或互動行為的卡片 / 區塊，一律不加 hover 效果**（不變色、不浮起、不變邊框、不放大）。

## 三、版面紀律（MUST）

具體「用多少」交給延伸決定，但「怎麼用才不亂」是護欄。

### 3.1 間距：一律 4px 倍數

禁止 5px、7px、10px、13px、25px 等非 4px 倍數值。基本階梯：4 / 8 / 16 / 24 / 32 / 48 / 64 / 96 / 128 / 160。

### 3.2 圓角：只用固定級距，且同頁一致

- 只能用 `0 / 2 / 4 / 6 / 8 / 12 / 16 / 24 / full` 這些級距（禁止 3 / 5 / 7 / 10 這種隨手填）。
- **同一頁的圓角必須一致**——不可按鈕 4px、卡片 8px、Modal 12px 混用。
- 確切用值：形象頁見形象頁延伸（現況 4px）、活動頁可依主題覆蓋但**仍須同頁一致**。

### 3.3 RWD 三段斷點

| 斷點 | 範圍 |
|---|---|
| mobile | `< 768px` |
| tablet | `768px ~ 1199px` |
| desktop | `>= 1200px` |

Mobile-first 寫法。字級各斷點放大基準：h1 32/40/48、h2 26/32/36、body 16/17/18。

### 3.4 同區塊對齊規則

**原則**：不同 section 之間容器寬度可以不一樣，但**同一個 section 內所有元素的容器寬度必須對齊一致**。

實作時自問 3 題：
1. 這 section 內用了幾種容器寬度？（理想 1 種，最多 2 種，第 2 種必須是「視覺層」如大圖、banner）
2. 同類元素（標題、內文、卡片、按鈕）左右邊界是否完全切齊？
3. 若不齊，差距 ≥ 64px（可接受，讀者理解為刻意）還是 < 32px（**「邊界靠近但沒對齊」，比明顯不同還醜，必須改**）？

→ Hero 全寬 + 內縮文字、大圖跨欄，屬「視覺層」特例，不算違規。

### 3.5 表格 mobile 卡片化

桌機保留完整 `<table>` 結構；mobile 一律**卡片化**（每 row 變獨立卡片、label/value 對照），不用橫向滾動。

## 四、文案品質（MUST）

### 4.1 禁用文案類型

| 禁用 | 說明 |
|---|---|
| 空洞行業術語 | 「賦能」「閉環」「抓手」「下沉」 |
| 過度稱讚式 | 「完美的」「極致的」「革命性的」 |
| 二分法行銷金句 | 「選擇 X 或 Y」「看見/看不見」「不只是 X 更是 Y」這類對仗式情緒口號，沒講具體事實 |
| 過長 | 標題 > 20 字、段落 > 4 行 |

→ 原則：**講具體事實、有可驗證的數字 / 動作**，而不是抽象口號塑造氣氛。

### 4.2 強調大字（font-size ≥ 22px 的獨立段落）

- 內文字數 **≤ 40 字**（含標點），超過就拆兩段或縮小字級。
- 加寬 letter-spacing（`0.04~0.08em`）與 line-height（`1.4~1.6`）。
- 每個 section 最多一個大字強調段落。
- **手動控制換行**：用 `<br>` 強制斷在逗點 / 句點後，**不可斷在詞中**（「不容妥協」四字須同一行）。

### 4.3 Statement Section 判斷（命中 3 條以上即違規禁用）

1. 整個 section 內只有一句話（無鋪陳、無圖、無列表）
2. 該句字級 ≥ 22px
3. 容器寬度 ≥ 880px
4. 上下 Section padding ≥ 96px
5. 內容是「空洞金句」（只有情緒口號、沒講事實）

→ **例外**：活動頁主視覺標語（如「2026 雙 11 限定」）若經明確標註為強化主旨可用，並在該檔專屬規範註記破例原因。

## 五、無障礙（MUST）

| 項目 | 標準 |
|---|---|
| 色彩對比 | 文字與背景 ≥ **4.5:1**（WCAG AA）|
| 字級最小 | 13px 為原則 |
| Focus 狀態 | 必須可見（input / button / link）|
| 觸控面積 | 可點元件至少 **44×44px** |
| 圖片 alt | 內容圖必填、裝飾圖 `alt=""` |
| 鍵盤導覽 | Tab 順序合理、Enter 可觸發按鈕 |
| 動畫 | 必須支援 `prefers-reduced-motion` 關閉 |

→ 上線前用 Lighthouse / axe 跑一次 A11y 檢查。

## 六、AI 友善（SEO / AEO）（MUST）

讓 AI 搜尋引擎（ChatGPT、Perplexity、Gemini 等）能正確理解、引用內容。核心三條：**結構清楚 + 內容明確 + 機器可讀**。

### 6.1 語意化 HTML

| 規則 | 說明 |
|---|---|
| 每頁一個 `<h1>` | 含核心關鍵字 |
| H2~H6 嚴格遞進 | 不跳級、不為視覺用 heading（視覺大字用 `<p class="text-large">`）|
| semantic tag | `<article>`/`<section>`/`<nav>`/`<main>`/`<header>`/`<footer>`/`<aside>` |
| 列表 `<ul>`/`<ol>`、表格 `<table>`、引述 `<blockquote>` | 不用 div 模擬 |

### 6.2 內容結構（AEO）

問答式（問題當 H2/H3）、倒金字塔（結論先行）、list/table 結構化、具體數據、避免空洞行銷。

### 6.3 Metadata

`<title>` 60 字內含關鍵字 + 品牌；`<meta description>` 155 字內；Open Graph 四項；Schema.org JSON-LD（商品 `Product` / 文章 `Article` / 組織 `Organization` / FAQ `FAQPage` / 麵包屑 `BreadcrumbList`）。

### 6.4 圖片 alt 與 Crawler

alt 描述內容（商品圖含品名 + 特徵；裝飾圖空字串；不堆關鍵字）；robots.txt 開放 `GPTBot`/`ClaudeBot`/`PerplexityBot`/`Google-Extended`/`CCBot`；sitemap.xml 每月更新；重要內容 SSR 直出 HTML、3 秒內首屏可見。

## 七、AI Slop 禁用清單（MUST，全頁面通用）

無論什麼主題都不准出現（活動頁再有趣也不例外）：

1. **滿版彩色漸層套版**（紫到藍、彩虹漸層等方向性套版；主題輔色的漸層須經 Owner 同意且不得走這幾種方向）
2. **圓角卡片 + 彩色左邊框 + icon + 標題 + 描述四件組**（一眼 AI Slop 模板）
3. **Emoji 當 UI 裝飾**（🚀✨💡 當項目符號、按鈕 icon、標題開頭）
4. **線條風格 SVG 插圖**（機器人、燈泡、齒輪等抽象示意）
5. **假數據過度整齊**（`$9,999`、`99.9% uptime`、`1,234 users`）
6. **整片深綠背景 + 白字**（戲劇化深綠 panel；淺灰 / 淺綠 OK）
7. **內文單字當裝飾字**（把單一字拉大當裝飾；改用 icon 或數字編號）
8. **不規則比例圖片裁切**（4:5 / 4:3 / 16:9 / 21:9 / 1:1 以外的奇怪比例）
9. **過度裝飾邊框**（雙線、虛線、漸層邊框）
10. **浮動 Tooltip**（除既有 hover 簡述外的彈出提示框）

## 八、表現層清單（護欄只管紀律，數值交給延伸）

以下面向的**確切值不在核心決定**——核心只保證「不管誰設值，都守住上面的護欄與下面的紀律」。這張表是三份文件的交界：形象頁延伸填「形象頁」欄的確切值，活動頁依「活動頁」欄的權限決定。

| 表現面向 | 核心紀律（MUST）| 形象頁 | 活動頁 |
|---|---|---|---|
| 圓角 | §3.2 固定級距、同頁一致 | 鎖定值（現況 4px）| **MAY** 依主題覆蓋，須同頁一致 |
| Section padding | 4px 倍數 | 鎖定值（桌機 64px）| **SHOULD** 用建議值，可依節奏覆蓋 |
| 容器 max-width | 同區塊對齊（§3.4）| 商品 880 / Blog 720 / 表單 524 | **MAY** 自訂 |
| 表單樣式 | 可辨識、focus 可見、iOS 不 zoom（16px）| 鎖定（框線式）| **MAY**（底線 / 框線皆可）|
| Icon | 線條粗細一致、繼承父層色 | 鎖定（Font Awesome，允許 fill）| **MAY** 換風格 |
| 懸浮元素（FAB/sticky CTA）| 不遮蓋關鍵內容 | 鎖定（保留 FAB / 錨點 / sticky CTA）| **MAY** 用或不用 |
| 詳情頁 layout | 響應式不破版 | 鎖定（桌機左圖右文 50/50）| **MAY** 自訂 |
| 圖文搭配 | 圖片 `<figure>` + alt 必填 | 鎖定（50/50、4:3）| **MAY**（比例、交替自訂）|
| 圖片按鈕組 | 整張 `<a>` 可點、hover 僅可點才有 | 鎖定（允許 overlay + zoom）| **MAY** 自訂變體 |
| Banner 比例 | 圓點在圖外、autoplay ≥ 5 秒、mobile 1:1 | 鎖定（桌機 16:9）| **SHOULD**（桌機建議 16:9~21:9）|
| Toast / Loading / 空狀態 | 錯誤不自動消失、可讀 | 鎖定（右上 / spinner / 有 icon）| **MAY** 自訂 |
| 動畫 | 節制、可被 reduced-motion 關閉（§五）| 鎖定（沿用既有 slider，不新增）| **MAY** 增加（scroll fade 等）|
| 攝影 / 素材風格 | 不踩 AI Slop（§七）| 極簡、自然光、低飽和 | **由每檔風格參考素材決定** |

→ 空白 / 未指定處：形象頁 fallback 到形象頁延伸的鎖定值；活動頁 fallback 到活動頁延伸的少數建議，其餘由風格參考素材補。

## 九、三層協作與衝突規則

### 9.1 一頁怎麼組出來

```
任何頁面 = 核心（本檔，全 MUST）
形象頁   = 核心 + 形象頁延伸（一組穩定鎖定值）
活動頁   = 核心 + 活動頁延伸（少數建議）+ 該檔風格參考素材（每檔外掛）
```

### 9.2 衝突時誰贏（優先序，高到低）

1. **核心護欄（MUST）** — 最高，任何情況不可破。素材 / 客戶要求違反護欄時，AI 要擋下並向使用者說明。
2. **該檔風格參考素材** — 活動頁專屬，覆蓋掉活動頁延伸的 SHOULD / MAY 建議。
3. **延伸的 SHOULD / MAY 建議** — 最低，僅在沒有更高層指定時作為 fallback。

→ 白話：**護欄永遠贏**；在護欄之內，**新給的素材贏過預設建議**（活動頁「新素材跟原建議衝突時，捨棄建議」就是這條）。

### 9.3 給 AI agent 的行為協議（活動頁）

做活動頁時，AI **MUST** 依序：

1. 先載入核心（本檔）作為不可違背的護欄。
2. 檢查是否已有該活動的**風格參考素材**（moodboard / 參考網頁 / 該檔期主色與元素）。
   - **沒有** → **主動提醒使用者提供**，並說明「活動頁的長相由素材決定，DS 只提供護欄與少數建議」。
3. 依「護欄 + 風格參考」推導出**該檔專屬規範**（專屬色、圓角、間距、動態量等 tokens），存到該活動資料夾 / LP brief，供本檔後續與同型活動複用。
4. 素材與活動頁延伸的建議衝突時，**以素材為準、捨棄建議**（但仍不可違反核心護欄）。

→ 此協議由 `ldp-ds` / `ldp-activity-strategy` 等 skill 實際觸發執行。
